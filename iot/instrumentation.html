Deep Instrumentation and Data Analytics

Background

Embedding multi-modal sensors in systems allows deep instrumentation of industrial and biological systems to capture fine-grained, real-time data across multiple spatial and temporal scales. Such sensor fusion then enables valuable descriptive and predictive analytics, and advanced AI applications.

Such instrumentation may be useful at different spatial or temporal scales. Micro-scale sensing may include in-body biosensors, wearables, or embedded MEMS for capturing localized physiological or material changes (e.g., glucose, vibration, pH, EMG). Similarly meso- or macro-scale sensors allow environmental, infrastructural or population-level monitoring. Similarly, on the time scale, sensing could be igh-Frequency, real-time telemetry (ECGs, accelerometers, vibration) or low-frequency/longitudinal sensing such as periodic glucose levels, weekly movement patterns, etc.

Despite differences in these scales, common engineering aspects apply to such systems. We focus here on two projects demonstrating the following:

  • Sensor fusion:
    • Heterogeneous sensor integration across modalities to enhance context awareness
    • Time sync and accurate timestamping for cross-correlation
    • Data Normalization & Calibration
  • Data Logging and Storage:
    • Edge-to-Cloud Pipelines to aggregate raw or preprocessed data for higher-order analysis.
    • Buffering and on-device storage to handle intermittent connectivity
  • Advanced Analytics and AI:
    • Time-series analysis and forecasting, e.g. predictig patient deterioration, maintenance needs, or energy consumption
    • Anomaly Detection for outliers in behavior, environment, or performance
    • Closed-loop control e.g. insulin pumps, climate control, autonomous navigation
The two projects highlighted below demonstrate successful implementation of these engineering principles with real, functioning prototypes.

Integrated device for multi-scale instrumentation

Sensing at multiple temporal and spatial scales often need to be combined into integrated, connected systems. Take for example an integrated system for storage and transportation of environment-controlled, perishable materials like vaccines. We may need to measure temperature and humidity at "lab/room scale", weather conditions at "city scale" to plan logistics and transport, and at "planet scale" (inclement weather, hurricanes, etc) that may dirupt the global supply chain.
Lab-scale data is best gathered with sensors, city-scale data is available through public APIs such as the OpenWeatherMap API, whereas planet-scale data is available from NASA's EONET (Earth Observatory Natural Event Tracker) system through NASA's open APIs. This is a good analogy for building systems that fuse data across sensors, medical systems, ERP systems, and so on.
This integrated system was architected as follows:

  • The base platform to integrate everything is the Adafruit Feather HUZZAH ESP8266, a microcontroller board that has rich connectivity functionality - not just for short-range / bluetooth but also WiFi so that we can access public APIs.
  • Using the Adafruit 2.4" TFT FeatherWing we a TFT display
  • Our lab-scale sensors include the DHT-22 for temperature and humidity, and an LDR (Light-dependent resistor)
    • ​
  • Our city-scale sensing comes from the OpenWeatherMap API
  • Our planet-scale sensing comes from the NASA EONET Open API
  • Finally, using the Arduino IDE, its rich suite of libraries, and opensource code we are able to integrate sensor data at all scales

Photosynthetic activity comparitive analysis

This project involves measuring the impact of multiple variables on photosynthesis. A terrarium is a closed glass enclosure containing soil, fertilizer and water that allows plants to grow in a measurable, controlled environment. Each terrarium is subject to different environmental conditions and the impact on plant growth was measured. A single integrated platform was built for each terrarium, comprising sensors for light intensity, temperature and humidity, and a data logger for timeseries data storage and analysis.
This integrated system was architected as follows:

  • To build the smallest possible form-factor, the Arduino Nano microcontroller was used
  • A TSL 2561 Lux sensor was used to measure light intensity variation.​
  • The DHT-11 sensor was used for integrated temperature and humidity.
  • An SD card module was used to log timeseries data in a simple, "flat" file system.
  • A portable power source was created using a DIY 4XAA battery case
  • Using the Arduino IDE, a data gathering loop was implemented for all sensors on syncronized time.
